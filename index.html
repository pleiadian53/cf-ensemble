    CF-Ensemble: Meta-learning via Latent-Factor-Based Collaborative Filtering¶

    A breakthrough framework for ensemble classification using collaborative filtering

    üåü Overview¶

    Ensemble learning combines multiple base models to improve predictive performance. This project introduces a novel ensemble transformation stage using latent factor-based collaborative filtering (CF) ‚Äì an additional layer of meta-learning that transforms base-level predictions before traditional ensemble integration.

    üí° The Core Idea¶

    We treat ensemble learning as a collaborative filtering problem:

    üéØ Why This Matters¶

    Classification in biomedical domains faces unique challenges: - ‚öñÔ∏è Class imbalance and skewed distributions - üîç Missing values and noisy measurements
    - üß¨ Complex biological relationships that vary by problem - üé≤ No consensus on best classifiers (problem-dependent)

    Our Solution: Transform ensemble predictions using matrix factorization to: 1. ‚ú® Increase reliability of probability estimates 2. üî¨ Discover patterns in how classifiers perform 3. üß≠ Interpret results through latent factor analysis 4. üéØ Identify challenging instances automatically

    üöÄ Quick Start¶

    Installation¶

    # Clone repository
git clone https://github.com/pleiadian53/cf-ensemble.git
cd cf-ensemble

# Create environment
mamba env create -f environment.yml
mamba activate cfensemble

# Install package
pip install -e .

    Basic Usage¶

    from cfensemble.data import EnsembleData
from cfensemble.optimization import CFEnsembleTrainer

# Your ensemble predictions (m classifiers √ó n instances)
R = ...  # probability matrix
labels = ...  # ground truth with NaN for unlabeled

# Train CF-Ensemble
ensemble_data = EnsembleData(R, labels)
trainer = CFEnsembleTrainer(latent_dim=10, rho=0.5)
trainer.fit(ensemble_data)

# Get improved predictions
P = trainer.predict(R)  # Reconstructed probabilities

    üéØ Features¶

    ‚úÖ Semi-Supervised Learning¶

    • Leverages unlabeled data to learn classifier reliabilities
    • No labels needed for calibration
    • Optimal at 5-10% minority class (validated!)

    ‚úÖ Confidence Weighting¶

    • Multiple strategies (uniform, certainty, label-aware, learned)
    • Handles systematic biases and miscalibration
    • Interpretable confidence weights

    ‚úÖ Optimized for Imbalanced Data¶

    • Best performance at 5% minority class (+3.94% PR-AUC gain)
    • PR-AUC as primary metric
    • Realistic biomedical scenarios (rare diseases, splice sites)

    ‚úÖ Dual Optimization Backends¶

    • ALS (Alternating Least Squares): CPU-friendly, stable
    • PyTorch: GPU acceleration for large-scale problems
